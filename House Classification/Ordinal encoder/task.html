Description

The type of the data encoder may greatly influence the performance of a machine learning model. It is always useful to try using various encoders and compare them, bearing in mind their respective data type.

Let's make another test to handle the categorical data â€” ordinal encoder. This encoder transforms arbitrary categories to integers, spanning from \(0\) to \(n-1\), where \(n\) is the number of unique feature values.

Let's take a look at the example of how OrdinalEncoder from scikit-learn works. Suppose you have the data, presented below:

Pet Owner Id
dog 1
cat 3
capybara 2

Ordinal encoder will transform the data to the following dataset:

Pet Owner Id Pet_encoded Owner Id_encoded
dog 1 2 0
cat 3 1 2
capybara 2 0 1

As you can see, the categorical variables represented by the numbers will be transformed to integers based on their ascending order. String categorical variables will be sorted in the alphabetical order and then assigned integers correspondingly.

Usually, this encoder is applied to data where the categories have known relationship, but why not try it?
